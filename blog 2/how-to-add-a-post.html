Publishing to The Long Field

A new piece every Sunday — here’s how, start to finish.

You never touch the design. Every post is a single plain-text file written in Markdown. You write it, drop it in a folder, add one line to a list, and it’s live. No code, no build step. Most weeks this takes about five minutes.

1

Start from the template

In the posts folder you’ll find _TEMPLATE.md. Make a copy of it and rename the copy to describe your post, using lowercase words joined by hyphens — no spaces, no capitals. This filename becomes the post’s web address (its slug).

# good filenames
the-art-of-the-unhurried-sunday.md
why-we-stopped-writing-letters.md
2

Fill in the details at the top

The block between the two --- lines is the post’s information. Fill each field in. Keep them on their own lines exactly as shown.

---
title: What a nervous system reset actually looks like
desk: Mindful
date: 2026-06-14
readingTime: 5 min
author: Zara
dek: And why your journal is the best tool for it.
hero:
---
titleThe headline. Sentence case — only the first word capitalised.
deskOne of the four pillars: Intentional, Mindful, Clarity, or Connected.
dateThe publish date, written year-month-day: 2026-06-14.
readingTimeA rough estimate, e.g. 5 min. (About 200 words a minute.)
authorWho wrote it. Usually Zara.
dekOne quiet sentence under the title — the promise of the piece.
heroOptional photo. Leave blank for now, or see step 5.
3

Write the piece below

Everything under the second --- line is your article. Write in plain Markdown — just type, leaving a blank line between paragraphs. The cheat-sheet at the foot of this page covers everything you’ll need: headings, a pull-quote, bold, italic, links.

Keep to the Marsh & Muse voice: calm, warm, assured. Say less, beautifully.

4

Add it to the list

Open posts/manifest.json and add your new filename — without the .md — to the top of the list, inside the quote marks. The top entry always becomes the lead story on the blog. Don’t forget the comma after it.

{
  "posts": [
    "the-art-of-the-unhurried-sunday",   ← your new post
    "what-a-nervous-system-reset-looks-like"
  ]
}
5

Add a photograph (optional)

If you have a hero image, put the file in the assets folder, then point the hero field at it. If you leave hero blank, a tasteful placeholder shows instead — the post still looks complete.

hero: assets/unhurried-sunday.jpg

Use warm, soft, natural-light photography — linen, eucalyptus, coffee, journalling. Landscape orientation works best.

6

Save, and it’s live

Commit your two changes — the new .md file and the updated manifest.json — to the repository. GitHub Pages publishes them automatically within a minute or two. That’s it. The post appears as the new lead, and the previous lead slides into the grid below.

A gentle rhythm

The blog promises readers a new piece every Sunday morning. Until your next post is published, the page shows quiet “Coming Sunday” cards in its place — so it never looks empty, and the promise is always kept. Aim to have each week’s post committed by Sunday morning, UK time.

Markdown cheat-sheet

Everything you can use in the body of a post.

## HeadingA section heading.
### Smaller headingA sub-heading.
**bold words**Emphasis. Use sparingly.
*italic words*A softer emphasis, or a quoted prompt.
> A line to featureBecomes a centred pull-quote.
- itemA bulleted list (one dash per line).
1. itemA numbered list.
[words](https://…)A link.
---Three dashes on their own line draw a thin divider.
One thing to know

This page and the blog need to be viewed on the live GitHub Pages site (or a local server) — not by double-clicking the file on your computer, which browsers block for security. On the published site it all works perfectly. If you ever want to preview before committing, ask a developer to run a local server, or simply commit and check the live page a minute later.
